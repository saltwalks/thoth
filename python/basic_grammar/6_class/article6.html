클래스

객체와 인스턴스

  1. 클래스A를 가지고 B를 만들었을때
    • B는 객체이다. 즉, 객체는 이놈의 특성이다
    • B는 A의 인스턴스이다. 즉, 인스턴스는 해당 객체가 어떤 클래스를 가지고 만들어진건지 나타내는 관계이다

클래스 기본구조 : class 클래스이름(상속받을클래스 - 없어도됨) :

  1. 괄호에 매개변수가 아닌 상속받을 클래스를 명시한다
  2. 콜론 이후 인덴테이션이 굉장히 중요하다

파이썬에서의 메소드 선언

  1. 기본구조 : def 메소드명(self, 매개변수1, 매개변수2, ... )
    • 파이썬은 메소드의 첫번째 인자로 무조건 이 메소드를 호출한 객체가 들어가게 되어 있다. 근데 대부분 자기자신이 호출하므로 관례적으로 self라는 이름을 쓴다
    • 객체를 이용해 메소드를 호출하는 경우 self에는 별다른 입력을 하지 않아도 된다(생략가능)
    • 하지만 메소드를 객체로 호출하는게 아닌 클래스로 호출하는 경우(예를들어 클래스.메소드() 이런식으로) 어떤 객체에서 호출했는지 알 수 없기 때문에 저부분에 호출하는 객체를 반드시 명시해줘야 한다

생성자

  1. 기본구조 : def __init__(self, 매개변수1, 매개변수2, ... )
  2. 파이썬에서는 생성자명으로 클래스 이름을 쓰는게 아닌 __init__라는놈을 사용한다

오버라이딩

  1. @override 이런거 필요 없이 그냥 자식에서 부모의 메소드를 재정의해주기만 하면 된다

클래스 변수

  1. 자바 클래스에서 변수 선언하듯이 쓰면 여기서는 클래스 변수로 선언된다
  2. 클래스 변수를 변경하는법 : 클래스명.변경메소드() 혹은 클래스명.변수 = 바꿀 값
  3. 주의할점 : 클래스를 통해 변경하는게 아닌 객체에서 변경을 하면 더이상 클래스변수로의 성질을 잃는다(주소값변경)
    • 클래스 변수를 변경하려면 객체단위에서 수정할게 아닌 상위의 틀인 클래스 단위에서 변경해라 뭐 이런뜻인듯
    • 객체를 생성하지 않고도 클래스 단위에서 변수나 메소드에 접근할 수 있기 때문에 가능한 일인듯

객체 변수

  1. 선언법 : 생성자(__init__)에서 self.변수 = 값 이렇게 선언해주면 객체변수로써 선언된다
  2. 물론 클래스의 아무 메소드에서도 self.를 붙이면 객체변수로써 선언되지만 필요한 객체변수는 생성자에서 다 생성하고 초기화해주는것이 통상적인 방법인듯
  3. self.를 붙이지 않으면 메소드 내에서만 사용 가능한 변수로써 선언된다
  4. 자바에서는 객체변수 선언부가 따로 있고 변수를 이용할때는 따로 this.같은걸 붙여주지 않아도 됐지만 파이썬에서는 이렇게 하더라
