예외처리

예외처리란

  1. 오류가 날 경우를 각각 조건문으로 처리해 줄 수도 있지만 같은 오류가 여러군데에서 발생할 여지가 있다면 이것을 한꺼번에 처리하는게 나을것이다

예외처리문의 기본 구조 : try : 내용 except 오류종류 : 대응

  1. try문을 실행하고 에러가 난다면 해당 에러에 대한 except문이 실행된다
  2. "내용" : 여기에 요류의 여지가 있는 코드를 적어준다
  3. "오류종류" : 여기에는 니맘대로 적는게 아니라 발생 가능한 오류 중 처리하고 싶은 오류를 적어준다
    • 오류도 하나의 클래스이다. 파이썬에서 미리 몇개의 오류 클래스를 선언해놓았고 나만의 오류를 만들 수도 있다
    • if문마냥 여러 오류에 대해 각각 except문을 지정해 놓을 수도 있다 - except문이 여러개여도 된다는 소리다
    • 에러종류를 명시하지 않으면 모든 에러에 대해 해당 except문이 걸리게 된다
    • 콤마(,)를 통해 여러 에러에 대한 대응을 하는 것도 가능하다 - 당연히 얘는 결과적으로 튜플로 하나로 묶이게 된다는것을 잊지 말자
    • IndexError : 리스트/튜플 등에서 선언되지 않은 인덱스를 참조할때
    • ZeroDivisionError : 0으로 어떤 수를 나누고자 할 때
    • FileNotFoundError : 존재하지 않는 file을 open하려고 할 때
    • 화면에 에러를 출력하고 싶으면 에러종류 as 변수로 해당 에러에 대응하는 문자열을 변수로 받아올 수 있다
    • 여러 오류를 튜플로 묶거나 오류를 명시하지 않아 여러 오류에 대해 처리할 때 as 변수를 쓰면 당연히 발생한 오류에 대해서만 오류문구가 변수에 담기게 된다
  4. "대응" : 해당 오류가 발생했을때 어떻게 대응할지를 적는 란이다.
    • 콜론 뒤에 적는 만큼 줄바꿈과 인덴테이션을 반드시 해줘야 한다
    • 대응에 아무것도 적지 않으면 실행시에 아무것도 안한다. "except 오류종류 :" 는 오류가 발생했을때 화면에 오류를 출력하기위한게 아니고 해당오류가 발생했을떄~는 뜻의 조건문개념이기 때문이다
    • 따라서 에러문구를 출력하고싶으면 print로 원하는 문구를 출력하던가 as를 통해 받은 에러문구를 출력할 수도 있다
    • 명시적으로 해당 에러에 아무런 대응도 하고싶지 않다면 pass 명령어를 활용하자
  5. finally : 오류가 발생해도 이놈만큼은 반드시 실행시키라는 뜻이다
    • 이놈은 except문 뒤에 같은 인덴트 레벨로 적어주는 것으로 보통 오류가 발생하면 프로그램이 종료되지만 finally문 만큼은 오류가 나도 여기에 있는 내용이 실행된다
    • 대표적 용례로는 file.close()와 같이 엮어 쓸 수 있다. 오류가 발생해도 file은 닫는게 추후 오류 방지를 위해 좋기 때문이다
  6. else : 오류가 발생하지 않으면 실행시키라는 뜻이다
    • try문을 다 실행했는데 오류가 안나면 else문을 실행하게 된다
    • 즉, 에러가 나면 except로 가지만 에러가 안나면 else문으로 가게 된다
    • 이것도 finally처럼 except문 뒤에 같은 인덴트 레벨로 쓰는 것이다

raise : 오류 일부러 발생시키기

  1. 오류를 일부러 발생시켜야 할 때도 있다. 다음은 그 용례이다
  2. if ~ raise # : 특정 경우에 대해 오류를 발생시키고 그것에 대한 대응을 except로 한번에 하고 싶을때 쓰면 된다. except로 예외처리를 하지 않고 모두 if문으로 처리한다면 같은 오류라도 그에 대한 똑같은 대응을 몇번이고 반복해서 기술해야 될 것이다
  3. 어떤 부모 클래스를 상속받는 모든 자식클래스에 대해 이 메소드는 반드시 오버라이딩했으렴 좋겠을 때 부모클래스의 해당 메소드에 raise로 오류를 발생시키면 된다. 그러면 이 메소드를 오버라이딩하지 않고 자식클래스에서 사용한다면 반드시 오류가 나게 될 것이다

Exception클래스 : 나만의 오류 만들기

  1. Exception클래스를 상속받으면 나만의 오류클래스를 만들 수 있다
  2. 기본구조 class 나만의오류(Exception): pass
    • 당연히 콜론 뒤에는 줄바꿈과 인덴테이션을 해줘야 된다
    • 안의 내용은 안적어도 된다. pass명령어를 사용한 것도 같은 이유이다
  3. 이렇게 나만의 오류를 만들고 나면 except문에 활용해도 문제없이 돌아가게 된다
    • 이렇게 나만의 오류를 선언하지 않고 except에 사용하게 된다면 정의되지 않은 오류로 에러가 발생하게 될 것이다
