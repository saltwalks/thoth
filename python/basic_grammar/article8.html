패키지

패키지 : 여러 파이썬 파일들을 종류별로 폴더로 묶어서 관리하는거

  1. 프로젝트가 커지면 코드가 길어져 가독성을 위해 파일을 여러개로 분리하게 되는데 파일이 또 너무 많아지면 정리가 안되기 때문에 종류별로 폴더로 묶어서 관리하게 된다

패키지 만들기

  1. 그냥 폴더별로 묶는다고 패키지가 완성되는게 아니다
  2. 각각의 패키지 폴더에 __init__.py 파일이 있어야 이것을 파이썬에서는 패키지로 인식한다
    • 물론 파이3에서는 안적어도 패키지로 인식하지만 하위버전호환을 위해 적어주는게 편하다
  3. 메인 실행 파일이 패키지의 최상단 디렉토리에 있으면 따로 path를 건드리지 않아도 되지만 그게 아니라면 앞서 배운대로 path를 수정해서 검색경로에 패키지 최상단 디렉토리를 추가해줘야 한다

__init__.py파일의 __all__변수

  1. from-import기능을 쓸때 import * 로 디렉토리의 모든 모듈들을 가져와도 실제로 실행해보면 에러가 난다
  2. 이것은 __all__변수가 설정되어있지 않기 때문
  3. __all__변수는 import *를 실행했을때 어떤 모듈을 가져오고 어떤 모듈을 가져오지 않을지 설정하는 리스트 변수이다
  4. 자료형이 리스트이기 때문에 __all__ = ["모듈1", "모듈2", ... ] 이런식으로 __init__.py파일에 추가해주면 import * 했을 때 저기애 추가되어있는 모듈들만 import할 수 있게 된다

상대경로

  1. 상대 경로를 이용하면 import시 길게 절대경로를 다 적어주지 않아도 된다
  2. 이것만 기억하면 된다 : .=현재 디렉토리, ..=부모 디렉토리
    • 디렉토리 A에 모듈 B가 있을 때 B의 부모가 A가 아니라는것을 주의하자
